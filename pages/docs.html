API 문서

MoaPay API는 RESTful 방식으로 설계되었습니다. JSON으로 요청하고 JSON으로 응답합니다.

빠른 시작

5분 안에 첫 번째 테스트 결제를 만들어보세요.

1. SDK 설치

# Node.js
npm install @moapay/node

# Python
pip install moapay

# Java (Maven)
<dependency>
  <groupId>dev.moapay</groupId>
  <artifactId>moapay-java</artifactId>
  <version>1.0.0</version>
</dependency>

2. API 키 설정

대시보드에서 발급받은 API 키를 환경변수에 저장합니다.

# .env
MOA_SECRET_KEY=moa_live_xxxxxxxxxxxxxxxxxxxx
MOA_WEBHOOK_SECRET=whsec_xxxxxxxxxxxxxxxxxxxx

결제 생성

POST/v1/payments

파라미터타입설명
amount필수integer결제 금액 (원 단위)
currency필수string통화 코드. 현재 KRW만 지원
method필수stringcard · kakaopay · naverpay · tosspay · bank · phone
order_idstring가맹점 주문 ID (중복 방지용)
customer_idstring구매자 식별자
metadataobject자유 형식의 추가 데이터
 
import Moa from '@moapay/node'
const moa = new Moa(process.env.MOA_SECRET_KEY)

const payment = await moa.payments.create({
  amount: 15000,
  currency: 'KRW',
  method: 'kakaopay',
  order_id: 'order_abc123',
  metadata: { product_name: '프리미엄 멤버십' }
})

웹훅 서명 검증

// Next.js App Router
import { headers } from 'next/headers'

export async function POST(req) {
  const body = await req.text()
  const sig = headers().get('moa-signature')

  const event = moa.webhooks.constructEvent(
    body, sig, process.env.MOA_WEBHOOK_SECRET
  )

  if (event.type === 'payment.completed') {
    await fulfillOrder(event.data.object)
  }

  return Response.json({ received: true })
}

에러 코드

MoaPay의 에러 메시지는 원인과 해결 방법을 포함합니다.

코드설명해결 방법
card_declined카드사 거절고객에게 다른 카드를 시도하도록 안내
insufficient_funds잔액 부족고객에게 잔액 확인 요청
invalid_card_number잘못된 카드 번호카드 번호 재입력 요청
expired_card만료된 카드유효기간 확인 또는 새 카드 등록
duplicate_order_id중복 주문 ID고유한 order_id 사용
